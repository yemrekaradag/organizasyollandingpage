# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

Marketing landing page for the **Organizasyol** app (an organization/event marketplace). Single-page React app built with Vite; all user-facing content is in **Turkish** — keep new copy in Turkish to match.

## Commands

```bash
npm run dev             # Start Vite dev server (http://localhost:5173)
npm run build           # Runs prebuild (sitemap) then vite build -> dist/
npm run preview         # Serve the built dist/ locally
npm run lint            # ESLint over **/*.{js,jsx}
npm run sitemap         # Regenerate public/sitemap.xml manually
npm run optimize:images # Re-run WebP optimization (needs cwebp installed)
```

There is **no test framework** configured (no test runner, no test files, no `test` script). Don't assume `npm test` exists.

`prebuild` runs [scripts/generate-sitemap.mjs](scripts/generate-sitemap.mjs) automatically before every `npm run build`, so `public/sitemap.xml` is regenerated from the static routes + blog content on each build.

## Architecture

- **Plain JSX, not TypeScript.** Files are `.jsx`/`.js`. `@types/react` is present in devDependencies but there is no `tsconfig.json` and no TS compilation — do not introduce `.ts`/`.tsx` without setting up the toolchain first.
- **Entry chain:** [index.html](index.html) loads [src/main.jsx](src/main.jsx), which wraps [src/App.jsx](src/App.jsx) in `<BrowserRouter>` and `<StrictMode>`.
- **Routing** (react-router-dom v7): route table lives in [src/App.jsx](src/App.jsx). `<Navbar>` and `<Footer>` render *outside* `<Routes>`, so they persist across every page; only the middle swaps. A `ScrollToTop` helper in App.jsx resets scroll to top on each navigation.
- **Page structure:** [src/pages/Home.jsx](src/pages/Home.jsx) is the landing page and composes the section components (`Hero`, `Features`, `FAQ`). The other pages under [src/pages/](src/pages/) are standalone Turkish legal/policy documents (Aydınlatma, Gizlilik, Kullanım, VeriIhlal, VeriSaklama) linked from the Footer. Adding a legal/policy page means: create the page in `src/pages/`, register a `<Route>` in App.jsx, and add a `<Link>` in [src/components/Footer.jsx](src/components/Footer.jsx).
- **Components** live in [src/components/](src/components/), each with a co-located `.css` file of the same name imported at the top of the component (e.g. `Hero.jsx` + `Hero.css`).

## Blog (markdown-driven)

- **Content lives in [src/content/blog/](src/content/blog/)** as `.md` files with YAML frontmatter (`title, slug, category, description, date, updated, author, keywords`). To publish a post, just drop a new `.md` file there — it's picked up automatically via `import.meta.glob` (eager, `?raw`) in [src/lib/blog.js](src/lib/blog.js). The `category` value must match a `slug` in [src/lib/categories.js](src/lib/categories.js) (the 15 fixed categories).
- **Frontmatter parsing is a small custom parser** in `blog.js` (no `gray-matter` — avoids a Buffer polyfill in the browser). Keep frontmatter to flat `key: value` lines; `keywords` is a comma-separated list. FAQ Q&A pairs are auto-extracted from the `## Sıkça Sorulan Sorular` section (`### question` + answer) to build FAQPage structured data — keep that exact heading.
- **Routes:** `/blog` (list), `/blog/kategori/:slug` (category landing, one per category), `/blog/:slug` (post). Rendering uses `react-markdown` + `remark-gfm` + `rehype-slug` + `rehype-autolink-headings`. Internal markdown links starting with `/` are rewritten to react-router `<Link>` (see `MD_COMPONENTS` in [src/pages/BlogPost.jsx](src/pages/BlogPost.jsx)).
- **Post covers are generated, not files** — [src/components/BlogCover.jsx](src/components/BlogCover.jsx) renders a per-category gradient + emoji, so no per-post image asset is needed.

## SEO

- **Per-page metadata uses React 19 native document metadata** via [src/components/Seo.jsx](src/components/Seo.jsx): it renders `<title>/<meta>/<link rel=canonical>` + Open Graph/Twitter + JSON-LD, which React hoists into `<head>`. **Every page/route must render one `<Seo>`** — it is the single source of truth (index.html intentionally has no per-page meta, only a fallback `<title>`). Don't add duplicate meta tags to index.html.
- **Structured data (JSON-LD)** builders are in [src/lib/structuredData.js](src/lib/structuredData.js) (`organizationLd`, `websiteLd`, `breadcrumbLd`, `blogPostingLd`, `faqLd`) — pass them to `<Seo jsonLd={...}>`.
- **Site constants** (`SITE_URL`, `SITE_NAME`, `DEFAULT_OG_IMAGE`) are exported from `Seo.jsx`; reuse them rather than hardcoding the domain.

## Images & fonts (performance)

- **All display images are pre-optimized WebP** in [src/assets/](src/assets/), generated by [scripts/optimize-images.sh](scripts/optimize-images.sh) (resize + `cwebp`). The original oversized PNG/JPEG masters are kept in the repo but **not imported** (so Vite doesn't bundle them). If you add a new screenshot, add it as a master then re-run `npm run optimize:images` and import the `.webp`. Always give `<img>` explicit `width`/`height`.
- **The LCP element is the first Hero phone screen.** [src/components/Hero.jsx](src/components/Hero.jsx) preloads it via a React 19 `<link rel="preload">` and uses `<AnimatePresence initial={false}>` so it paints immediately. Do not reintroduce eager-loading of all carousel images.
- **The Outfit font is self-hosted** via `@fontsource-variable/outfit` (imported in [src/main.jsx](src/main.jsx)); there is no render-blocking Google Fonts link. The CSS family is `"Outfit Variable"`.

## Styling

- **Vanilla CSS only** — no Tailwind, no CSS-in-JS. Styling is split between co-located component CSS files and global rules.
- **Design tokens** are CSS custom properties defined in `:root` in [src/index.css](src/index.css) (purple brand palette, glassmorphism vars, gradients, Outfit font). Reuse these variables instead of hardcoding colors.
- **Dark mode only** — `color-scheme: dark` is set globally; there is no light theme.
- Shared utility/global classes referenced across components include `.container` (max-width wrapper), `.text-gradient`, `.glass-panel`, and `.btn-primary` / `.btn-sm`.
- **Animations** use `framer-motion` (`motion`, `AnimatePresence`). Icons come from **both** `lucide-react` and `react-icons` (e.g. `FaInstagram`).

## Deployment

- [Dockerfile](Dockerfile) is a multi-stage build (node:20-alpine → nginx:alpine).
- **Gotcha:** the Dockerfile expects this project nested under a `web/` directory (`COPY ./web/...`), so `docker build .` from the repo root will fail as-is — it's written for a parent monorepo layout. The nginx config it references now exists at [nginx/conf.d/default.conf](nginx/conf.d/default.conf).
- **SPA routing on the server:** the nginx config uses `try_files $uri /index.html`, which is required so deep links to blog URLs (e.g. `/blog/dugun-organizasyonu-rehberi`) return 200 instead of 404. Any other host serving `dist/` must do the same fallback.
- `public/robots.txt` and the build-generated `public/sitemap.xml` both reference `https://organizasyol.com/`. The OG fallback image is `public/og-image.png` (1200×630).
